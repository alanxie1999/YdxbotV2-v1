# 功能概览

这页只讲一件事：

**这个脚本到底能做什么。**

## 一、自动接收消息

脚本会持续监听：

- 盘口触发消息
- 结算结果消息
- 红包/附加消息（如当前配置启用）

这样你不需要一直盯着群消息判断什么时候该操作。

## 二、自动下注

脚本会按当前预设：

- 计算下一手金额
- 判断下注方向
- 组合按钮点击
- 等待结算回写

对长期手动押注的人来说，最直接的价值就是：

- 少重复点按钮
- 少手动算下一手金额
- 少因为节奏快而漏单或按错

## 三、简单跟随策略

当前版本使用简化的跟随策略：

**默认跟随**：
- 上一手开 1（大）→ 下一手押大
- 上一手开 0（小）→ 下一手押小

**交替打破**：
- 检测到 10101 或 01010 纯交替模式
- 第 6 手反向打破（10101→押小，01010→押大）
- 出现连续相同结果后自动回到跟随模式

这种策略的优势是：
- 逻辑简单，易于理解
- 执行速度快，延迟低
- 不依赖外部 API，稳定性高

## 四、风控与暂停

脚本支持多种运行保护：

- 连输告警
- 盈利达标暂停
- 炸号保护暂停
- 资金不足暂停
- 模型异常暂停
- 连续观望暂停

这些保护的作用不是“让脚本更激进”，而是让你更清楚什么时候该停、什么时候能继续。

## 五、状态与通知

脚本会持续给出：

- 状态面板
- 下注执行通知
- 结算结果通知
- 连输告警
- 连输结束通知
- 启动成功通知
- 更新与回退结果通知

你可以把这些消息发到：

- 管理员控制台
- Telegram 通知 Bot
- IYUU

## 六、多账号运行

脚本支持多账号独立运行。

每个账号可以独立配置：

- session
- cookie / x_csrf
- 管理员入口
- 通知渠道
- AI key
- 预设
- 状态文件

这样适合：

- 一个人维护多个账号
- 或同一台机器上分开管理不同账号

## 七、命令控制

脚本支持通过管理员入口进行日常控制，例如：

- 启动预设
- 查看状态
- 暂停 / 恢复
- 刷新余额
- 查看预设
- 设置本轮目标金额
- 更新 / 回退 / 重启

## 八、适合哪些人

这个脚本尤其适合：

- 已经长期手动盯盘的人
- 对下注节奏和按钮操作很熟，但不想重复机械操作的人
- 希望把多账号值守收进统一控制面的使用者
