# 连输止损后自动重置功能

## 功能说明

当连输达到预设的止损手数时，系统会自动：

1. **暂停 10 局** - 等待 10 轮开奖
2. **重置状态** - 清零连输计数和倍投进度
3. **首注重启** - 用预设的首注金额重新开始下注

## 示例流程

假设预设参数：`/ys 2w 1 10 3.0 2.5 2.2 2.1 20000`

- 首注：20,000
- 止损：10 手

### 正常流程

```
第 1 手：押 20,000 → 输
第 2 手：押 60,000 → 输  (20,000 × 3.0)
第 3 手：押 126,000 → 输 (60,000 × 2.1)
...
第 10 手：达到止损上限
```

### 触发止损后

```
⚠️ 已达到预设连投上限，已自动暂停
当前预设最多连投：10 手
当前连输：10 手
等待 10 局后将用首注金额重新开始

[系统暂停 10 局]

第 1 手：押 20,000 → 重新开始  (首注重置)
第 2 手：押 60,000 → 如继续输
...
```

## 优势

1. **保护资金** - 避免连续亏损导致资金大幅缩水
2. **冷静期** - 10 局暂停让系统观察盘面变化
3. **重新开始** - 用首注金额重启，降低风险
4. **自动执行** - 无需手动干预，全自动流程

## 修改配置

### 调整暂停局数

编辑代码 `zq_multiuser.py:4168`：

```python
rt["stop_count"] = 10  # 修改为想要的局数
```

### 调整止损手数

使用 `/ys` 命令：

```bash
# 格式：/ys 名称 连续 止损 一输 二输 三输 四输 首注

# 设置止损为 8 手
/ys 2w 1 8 3.0 2.5 2.2 2.1 20000

# 设置止损为 12 手
/ys 2w 1 12 3.0 2.5 2.2 2.1 20000
```

## 通知消息

触发止损时会收到通知：

```
⚠️ 已达到预设连投上限，已自动暂停
当前预设最多连投：10 手
当前连输：10 手
等待 10 局后将用首注金额重新开始
```

## 如何确认已重置

使用 `/status` 命令查看状态：

- `bet_sequence_count: 0` - 倍投进度已重置
- `lose_count: 0` - 连输计数已清零
- `bet_amount: [首注金额]` - 下注金额回到首注

## 注意事项

1. **资金充足** - 确保菠菜资金足够支持新的倍投链
2. **耐心等待** - 暂停期间不要手动干预
3. **观察盘面** - 利用暂停期观察盘面走势
